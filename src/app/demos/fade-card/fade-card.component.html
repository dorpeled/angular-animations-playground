How it works:

The fadeAnimation trigger with disabled initial animations:

  • :enter transition with opacity 0 to 1
  • :enter transition with height 0 to auto
  • :leave transition with opacity 1 to 0
  • :leave transition with height auto to 0
  • Overflow hidden during animations to prevent content spillage
  • 500ms duration with ease-in/ease-out timing functions
  • Animation disabling feature to skip initial animations
  • Animations enabled after initial rendering using afterNextRender

This technique prevents animations on page load, but allows smooth expanding/collapsing animations when adding/removing cards later.
